# Data Sources

Provenance for every external dataset used to bootstrap Sankalp's mock
electoral layer. See `docs/DATA.md` §3 for the policy: every datum
either has a verifiable public source, or is flagged synthetic in-line.

Last verified: 2026-05-04 (Phase 1 ship).

## Constituency master (`ac_master.csv`)

Hand-curated list of 100 representative Vidhan Sabha + Lok Sabha
constituencies covering 28 jurisdictions. Names, codes, district
mappings, and Lok Sabha containment derive from:

- Election Commission of India — *Atlas of Indian Elections* (2019).
  https://eci.gov.in (Statistical Reports → Atlas)
- Delimitation Commission of India — Delimitation Order, 2008.
  Gazette of India archives, Ministry of Law and Justice.
- State Chief Electoral Officer (CEO) websites for boundary updates
  post-2014 (Telangana, J&K reorganisation).

Total electors / male / female counts are seeded from ECI Form-22A
publications (state-level rolls as of latest available revision)
where verifiable, and synthesised within ±10% of the publicly
reported district elector total otherwise. The Provenance block on
each Constituency record discloses this.

## Election history (5 elections per AC)

All 500 election records (100 ACs × 5 elections) ship with
`is_synthetic_election: true` and a `synthesis_note` until each
record is hand-checked against the cited ECI Statistical Report.
Anchor ACs (KA-151 Bommanahalli, BR-180 Patna Sahib, TG-064
Goshamahal, WB-159 Bhabanipur, MH-176 Worli) carry headline values
(year, type, winner, runner-up) that match commonly reported public
outcomes; precise margins and turnout pending verification.

Source format: `ECI Statistical Report {state} {year}, AC {ac_code}`

These reports live at:
https://eci.gov.in/statistical-report/statistical-reports/
(state-by-state, election-by-election PDFs.)

## Booth metadata

~510 booths across the 100 ACs. Booth names follow real ECI
naming conventions ("Government Higher Primary School, ...") drawn
from state CEO published booth lists. Booth addresses and geo
coordinates are synthesised within ±0.02 deg of the AC centroid.

Source format: `State CEO {state_name}` per `Booth.source`.

State CEO portals (sample):
- Karnataka: https://ceo.karnataka.gov.in
- Bihar: https://ceobihar.nic.in
- Telangana: https://ceotelangana.nic.in
- Maharashtra: https://ceo.maharashtra.gov.in
- West Bengal: https://ceowestbengal.nic.in

## Booth accessibility flags

**Fully synthetic.** ECI does not publish booth-level accessibility
data publicly. Per `docs/DATA.md` §4, every `AccessibilityFlags`
block carries `synthetic: true` and Sankalp's UI surfaces a
"demo data" chip on every booth card.

Distribution seeds: 62% wheelchair, 71% ramp, 83% ground-floor,
18% sign-language, 7% braille ballot. Plausible defaults based on
the Persons with Disabilities Act 2016 and ECI's accessibility
guidelines.

## Voter records

**Fully synthetic.** No real voter is in this dataset. 4,500 records
generated by Faker (Indian locale `en_IN`, fixed seed=42).

- Names: Faker first/last names, weighted across genders.
- EPIC numbers: 3-letter state-derived prefix + 7 digits, format
  per ECI Form-001 specification (no checksum verification —
  ECI's actual checksum algorithm is non-public).
- Addresses: real PIN codes, real localities (drawn from the
  AC's `ac_name`), invented house numbers and street names.
- DOB: uniform in [18, 92] years from 2026-01-01 reference.
- Demo personas (Riya, Ravi, Priya, Anjali x2): scripted into
  the roll so smoke tests find them deterministically.

Every record has `is_synthetic: true`.

## Demographics

Literacy and urban% fields are synthesised within plausible bands
per state. Real Census 2011 district-level values
(https://censusindia.gov.in/) can be substituted by editing the
build script — for v0 we mark `is_synthetic: true` on every
Demographics block and source it as
`Synthesis vSANK-1; cross-check against Census 2011 + ECI Atlas`.

## What is NOT scraped

`voters.eci.gov.in` is **never** hit programmatically by Sankalp.
The portal does not expose a public API and scraping arguably
violates the Terms of Service. No Sankalp tool, agent, or build
script makes a request against that host. See `docs/DATA.md` §1.
